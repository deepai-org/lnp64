# Theorem/RTL Coupling Evidence Index

This index is the human-readable view of
`formal/theorem_rtl_coupling_manifest.json`. It is intentionally conservative:
the current claims remain T1 because their RTL coupling still goes through
assertions, executable-model traces, and Docker gates. The M1 capability slice
now has schema-backed typed cap commit and post-state projection records, a
typed RTL state-projection boundary, local typed-commit assertions, an explicit
RTL projection/refinement-shape relation, Lean packed-schema mirrors and RTL
packed typedef layouts checked against the shared RTL schema, and an executable pre/commit/post refinement comparison; M2
gate/continuation, M4 VMA/MMU, M5 DMA, M7
waitable/scheduler, and M14 Resource Domain slices now have transition-invariant
Lean evidence. These claims are still not T4: M1's emitted packed-bit trace
check is executable/Python-mirrored evidence, not a formally checked or
generated RTL-to-Lean bit-refinement artifact, and the rows are not full
shared-schema T2 evidence across the whole architectural claim.

The active work-order gate is M1 first: do not treat later transition-invariant
slices as the template until M1 has a credible refinement-coupled shape. For
M1, typed traces, local assertions, and executable comparison are progress
toward T4; they are not a substitute for a formal/generated RTL-to-Lean
refinement artifact that maps every accepted M1 commit and authority-bearing
post-state projection to an allowed Lean transition.
