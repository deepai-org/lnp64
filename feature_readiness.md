# LNP64 Feature Readiness Ledger

This is the informal readiness table for features that cross the Rust emulator,
RTL, software toolchain, and proof tracks. It is not a completion claim. A row is
ready to grow only when it names the touched object, the owning engine, the
authority source, the generation or epoch that rejects stale use, the trace
record, and how emulator/RTL comparison can observe the behavior.

Status vocabulary:

- `done`: checked artifact exists in the repository.
- `partial`: useful artifact exists, with an explicit gap in the row.
- `planned`: named target exists, but the implementation or gate is not active.
- `blocked`: cannot be claimed until the blocker in the row is removed.
